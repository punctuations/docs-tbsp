# Generating messages
The generate command generates a commit message, but **does not commit it**.

This command has five possible flags to use:
- `--full` (`-f`): full length commit description, showing all files changed.
- `--unstaged` (`-u`): generate a commit message for unstaged files.
- `--no-comment` (`-n`): prompts the user for a short description instead of using comments.
- `--select` (`-s`): allows user to choose the file to showcase in the short commit message.
- `--co-author` (`-c`): adds a co-author to the message.

## Full generated message
`tablespoon generate --full`

```
✔ feat
feat(cmd/generate.go): overhaul to foobar


- cmd/generate.go - 8 changes
- README.md - 3 changes

Authored-by: matt <matt@dont-ping.me>
```

## Unstaged generated message
`tablespoon generate --unstaged`

```
✔ feat
feat(cmd/generate.go): overhaul to foobar
```

## No-comment generated message
`tablespoon generate --no-comment`

```
✔ feat
What was changed in references/generate.mdx?: example comment
feat(references/generate.mdx): example comment
```

## Select generated message
`tablespoon generate --select=README.md`

```
✔ feat
docs(README.md): add better documentation
```

## Co-author generated message
co-author requires the `--full` flag to work properly.

`tablespoon generate --co-author=punctuations --full`

```
✔ feat
feat(cmd/generate.go): example comment


- cmd/generate.go - 9 changes

Authored-by: matt <matt@dont-ping.me>

Co-Authored-by: punctuations <punctuations@users.noreply.github.com>
```
